bhop script

!Скрипт, который позволяет быстро и без потери скорости совершать Bunnyhop, выполняя команды +jump через каждый второй кадр. Важно знать, что этот скрипт обеспечивает плавное прыжки кроликом только при определенных значениях fps_max, потому что при некоторых частотах кадров время, которое игрок проводит в воздухе, может привести к тому, что игрок приземлится на землю, пока скрипт выполняет -прыжок. Дополнительную информацию по этой теме можно найти на странице «Эффекты FPS».!

alias bh1 "+jump;alias bh@ bh2;special"

alias bh2 "wait;alias bh@ bh3;special"

alias bh3 "-jump;alias bh@ bh4;special"

alias bh4 "wait;alias bh@ bh1;special"

alias +bhop "alias _special bh@;bh1"

alias -bhop "alias _special;-jump"

bind SPACE +bhop

Duckroll script

!Скрипт, который делает непрерывные двойные прыжки (небольшие псевдопрыжки, которые поднимают вас в воздухе примерно на 32 единицы) чрезвычайно быстро. В основном он используется для контроля скорости при движении на очень высокой скорости и для поддержания скорости при движении по направлению к лестнице. Его также можно использовать для поддержания скорости при игре в версию Half-Life, в которой скорость Bunnyhopping ограничена из-за того, что ограничитель скорости Bunnyhop вообще не обнаруживает даблдаки. См. duck!

alias +duckroll "alias _special duckroll;duckroll"

alias -duckroll "alias _special"

alias duckroll "+duck;wait;-duck;wait;special"

Double-Duck script

Простой сценарий, который выполняет двойное приседание, лучше всего работает при выполнении техники «Счет-прыжок» или при быстром подъеме по ящикам. Используйте bind +dd

alias +dd "-duck;wait;+duck;wait;-duck;wait;+duck"

alias -dd -duck

Jumpbug script

Это поможет вам при попытке выполнить Jumpbug. Просто удерживайте клавишу, чтобы присесть, и отпустите ее, чтобы мгновенно присесть и прыгнуть. Помните, что это не облегчает выполнение jumpbug. Это всего лишь привязка, которая поможет вам. Используйте bind + jumpbug

alias _jumpbug

alias _jumpbugfunc "-duck;+jump;w5;-jump"

alias +jumpbug "+duck; alias _jumpbug _jumpbugfunc"

alias -jumpbug "_jumpbug; alias _jumpbug"

Usespam script

Сценарий, который очень быстро спамит команду +use. Обычно он используется для быстрого нажатия кнопок, но при определенной технике его можно использовать для создания псевдо-объектов, которые уносят с собой объект, которым вы манипулируете. Используйте bind <ключ> +usespam

alias +usespam "alias _special usespam;usespam"

alias -usespam "alias _special"

alias usespam "+use;wait;-use;wait;special"

Quick-Use script

Простой скрипт, выполняющий +use в один кадр, не замедляющий вашу скорость, идеально подходящий для нажатия кнопок, когда вы прыгаете рядом с ними. Используйте bind quse

alias quse "-use;wait;+use;wait;-use"

Gauss Boost script

Скрипт Gauss-jumping для мультиплеера. Основное отличие состоит в том, что он подталкивает вас в направлении, к которому вы стремитесь, чтобы воспользоваться тем фактом, что вы способны повышать Гаусса по вертикали. Используйте bind +tau

alias _taubo

alias _taubofunc "cl_pitchup 180;cl_pitchdown -180;-attack2;wait;cl_pitchup -12;cl_pitchdown 12;wait;cl_pitchup 89.999;cl_pitchdown 89.999"

alias +tau "+attack2; alias _taubo _taubofunc"

alias -tau "_taubo; alias _taubo"

Object Boosting script

В сценариях ускорения объектов число представляет скорость (в единицах в секунду), которую вы получите, если будете выполнять их со скоростью 100 кадров в секунду. Обратите внимание, что у вас могут быть разные результаты при разных значениях FPS из-за повышенного ускорения при ускорении объектов. Используйте bind obbo

alias obbo2000 "+use;w11;-use;+jump;w;-jump"

alias obbo1500 "+use;w10;-use;+jump;w;-jump"

alias obbo1300 "+use;w9;-use;+jump;w;-jump"

alias obbo1000 "+use;w8;-use;+jump;w;-jump"

alias obbo800 "+use;w7;-use;+jump;w;-jump"

alias obbo600 "+use;w6;-use;+jump;w;-jump"

alias obbo500 "+use;w5;-use;+jump;w;-jump"

Wait Table

Если вы используете Bunnymod XT, таблица ожидания не обязательна и может быть заменена эквивалентной командой w <номер> (обратите внимание на пробел между ними). Преимущество этого подхода заключается в возможности использовать произвольное количество ожиданий в одной команде, например w 3528, вместо того, чтобы использовать w1000;w1000;w1000;w500;w28, если бы использовалась таблица ожидания. Если вы не используете Bunnymod XT, следующая таблица ожидания может быть плохой заменой w. Эта таблица псевдонимов генерирует в общей сложности 110 псевдонимов ожидания, что отлично подходит для простого создания сложных сценариев.
